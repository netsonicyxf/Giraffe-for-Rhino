Giraffe Guide

Abstract

Imports wireframe geometry as structural model into SOFiSTiK.

The Layer Structure

Giraffe organizes structural information using a strict layer structure:

  • all input geometry in a main 'input' layer
  • 1st children layers: 'nodes', 'beams', 'trusses', 'cables', 'springs'
  • 2nd children layers: ' // group number // [ // structural properties // ] { // description // }' format
  • 3rd, 4th etc. children layers: '// structural properties // { // description // }'

Layers can be nested infinitely. Note that structural properties should only be in square brackets if the a group number is not specified. In fact, Rhino does not allow layer names to start with square brackets.

The nodes sublayer should not have further sublayers.

example

layer: 2 [ncs 5 ahin mymz] {main facade girders}

within 'beams' layer will produce statement:

!#!Label grp 2 - main facade girders

beam prop ncs 5 ahin mymz

rules:

  • leading and trailing whitespaces allowed
  • group number is mandatory
  • layer properties may be omitted, including square brackets

Custom Properties

By adding custom names to structural elements, their properties may be modified.
